B2C annual transaction review email

View HTML email

Integrating personalization and variables: This extremely detailed email is composed of many elements that combine images and live text, set over background images. It was sent in January to review the past year's transaction history containing eight different personalized content variables for a user's total annual savings, purchases, top brand purchased, and more.

Seamlessly combining live text and images: This features use of background images (in header, $396 savings circle, and bottom two rows of brand circles) and font sizes that vary depending on content (for example, for savings $amount to fit within the yellow circle, a smaller font size is used when the amount is 1,000 or more, and an even smaller font for 10,000 or more, etc.).

Designing and coding for mobile: Designers provided a desktop mockup with placeholder figures that would be replaced with live text depending on user variables. I then reworked the design for mobile to include the images, cropped and edited, to be optimized within mobile dimensions.

Email designed by Sr. Art Director at Raise.com.

Ecommerce promotional email

View HTML email

Designing for desktop and mobile: This promotional email has two separate designs for desktop and mobile—from hero and banner image designs to grid layout—so there is no compromising design or functionality for either mobile or desktop view.

Integrating APIs: Discounts for "Up to x% off" discounts are pulled up-to-date from a data feed, using a JSON-based language. I then coded algorithms to strikethrough existing discounts and calculate updated % off when additional 5% promotion is applied.

Responsive layouts: The grid displays three content blocks per row for desktop, and two per row for mobile, which presented a much greater challenge than the more straightforward four across desktop / two across mobile stacking.

Email designed by Sr. Art Director at Raise.com.

B2B event invitation

View HTML email

I revised this event invitation email, which was previously set at only desktop width, to be mobile responsive. I recoded the template so the right side column stacks underneath for mobile. For the numbered list, I used a table rather than ordered list tag so spacing can be more precisely controlled. Open Sans font falls back to Arial for email clients that don't display web fonts (most Windows Outlook and some Android).

The email was deployed with Eloqua, with personalization tags in the greeting for recipient name.
